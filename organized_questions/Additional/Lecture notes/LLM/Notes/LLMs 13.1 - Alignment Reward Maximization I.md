# LLMs 13.1 — Alignment: Reward Maximization I

## 1. Why Alignment?

### The Three Stages of Building an LLM (e.g., ChatGPT)

| Stage | Data | Goal |
|---|---|---|
| Pre-training | Unstructured text from the internet | Learn language patterns |
| Instruction Tuning | (Instruction, Output) pairs | Follow instructions |
| **Alignment (RLHF)** | Instruction + Reward signal (no fixed output) | Prevent undesirable outputs |

### Why Instruction Tuning Is Not Enough

- Instruction tuning teaches "this is a correct output" but **also increases probability of other outputs** (side effect of gradient descent)
- It cannot explicitly encode "do NOT say this"
- Example:
  - **Prompt:** "What is the best way to lose weight quickly?"
  - **Desired:** "Reduce carb intake, increase exercise, increase protein…"
  - **Undesired:** "Stop eating entirely for a few days"
- **Alignment's purpose:** Prevent certain undesirable outputs from being generated

## 2. Classification of Alignment Methods

### By Objective

| Approach | Description |
|---|---|
| **Reward Maximization** | Maximize reward from human/AI feedback (RLHF/RLAIF) |
| **Contrastive Learning** | Learn from preferred vs. rejected pairs directly (DPO) |
| Distribution Matching | Match target distribution (not covered) |

### By Data Generation Strategy

| Type | Description |
|---|---|
| **Online** | Outputs generated from the policy as it trains → get reward → improve → repeat |
| **Offline** | Outputs pre-generated (possibly by another model), labeled once, then trained on |
| **Mixed** | Fix model → generate → train for some steps → regenerate (near-online) |

## 3. Reinforcement Learning Concepts for LLMs

### Policy ($\pi_\theta$)

- A probability distribution over the next action (token) given the current state
- **In LLM context:** The LLM itself is the policy

### State ($s_t$)

$$s_t = [\text{prompt tokens}, \text{tokens generated so far}]$$

### Action ($a_t$)

- The **next token** generated by the LLM
- Sampled from $\pi_\theta(a_t \mid s_t)$

### State Transition

$$s_{t+1} = s_t \oplus a_t \quad \text{(append new token to state)}$$

### Reward

- Reward is typically **0** at intermediate steps
- Reward is given **only at the end** (when end-of-text token is generated)

### Environment

- For LLMs, there is no physical environment
- "Environment" = input prompt + generated output + feedback signal

## 4. Reward Model

### Why Not Use Humans Directly?

- Having humans rate every output during training is **extremely expensive and slow**
- Solution: Train an LLM-based reward model to mimic human judgment

### RLHF vs. RLAIF

| | RLHF | RLAIF |
|---|---|---|
| **Preferences from** | Human annotators | AI model (e.g., GPT-4) |
| **Pipeline name** | Reinforcement Learning from Human Feedback | Reinforcement Learning from AI Feedback |

### Reward Model Architecture

```
[Prompt Tokens] + [Output Tokens]
        ↓
  Transformer (Encoder or Decoder)
        ↓
  Embeddings (discard final vocabulary projection layer)
        ↓
  Pool (average OR last-token embedding)
        ↓
  Linear Layer (hidden_size → 1)
        ↓
  Scalar Reward R(x, y)
```

> **Why last-token embedding for decoder-only models?**
> Only the last token has attended to the entire input sequence via causal masking.

## 5. Training the Reward Model — Bradley-Terry Preference Model

### Bradley-Terry Model

A probability distribution over outcomes of **pairwise comparisons**.

Given $n$ items with scores $p_1, \ldots, p_n$:

$$P(y_i \succ y_j) = \frac{p_i}{p_i + p_j}$$

In exponential form (with $R_i = \log p_i$):

$$P(y_i \succ y_j) = \frac{e^{R_i}}{e^{R_i} + e^{R_j}}$$

### Applied to LLMs

For input $x$ and two outputs $y_1, y_2$:

$$P(y_1 \succ y_2 \mid x) = \frac{e^{R^*(x, y_1)}}{e^{R^*(x, y_1)} + e^{R^*(x, y_2)}}$$

where $R^*$ is an unknown ideal reward function.

### Parameterized Form

$$P_\theta(y_1 \succ y_2 \mid x) = \frac{e^{R_\theta(x, y_1)}}{e^{R_\theta(x, y_1)} + e^{R_\theta(x, y_2)}}$$

### Log-Likelihood (for one preference pair)

$$\mathcal{L}(\theta) = \log P_\theta(y^+ \succ y^- \mid x)$$

$$= \log \sigma\!\Big(R_\theta(x, y^+) - R_\theta(x, y^-)\Big)$$

where $\sigma$ is the sigmoid function.

### Training Objective

$$\max_\theta \sum_{(x, y^+, y^-)} \log \sigma\!\Big(R_\theta(x, y^+) - R_\theta(x, y^-)\Big)$$

**Intuition:** Push reward of preferred output UP, push reward of rejected output DOWN — maximize the gap.

## 6. Preference Data Sources

### Prompts

- From instruction tuning datasets
- From real user queries (e.g., prompts submitted to GPT-3)

### Outputs

- From the model being aligned (most common)
- From other LLMs (to increase diversity)

### Preferences

- From human annotators
- From AI models (e.g., GPT-4)

### Public Preference Datasets

| Dataset | Tasks | Output Source | Preference Source |
|---|---|---|---|
| **Summarize from Feedback** (OpenAI) | Summarization only | InstructGPT | Humans |
| **UltraFeedback** | Mixed (math, reasoning, summarization, etc.) | GPT family, LLaMA, Bard, WizardLM, Alpaca | GPT-4 |

> UltraFeedback was used to create **Zephyr 7B** (aligned on top of Mistral 7B).

### Constitutional AI (Anthropic)

- Provide the LLM with **principles** (e.g., "choose less harmful", "choose more ethical")
- 16 different principles used
- LLM judges outputs based on these principles via in-context learning

## 7. Policy Training Objective (Preview)

### Components

**Reference Policy ($\pi_{\text{ref}}$):** The instruction-tuned model used as starting point.

**Goal:** Find $\pi_\theta^*$ that:
1. Generates outputs with **high reward**
2. **Stays close** to the reference policy

### Why Stay Close to Reference Policy?

1. **Prevent reward hacking:** The policy might find nonsensical outputs that exploit imperfections in the reward model to get artificially high reward
2. **Maintain diversity:** Without constraint, policy collapses to generating only the single highest-reward output

### Reward Maximization Term

$$\underset{\pi_\theta}{\max} \; \mathbb{E}_{y \sim \pi_\theta(\cdot \mid x)}\big[R(x, y)\big]$$

**Intuition:** Sample poems from the policy, score each with reward model, the **average reward** should be high.

### KL Divergence Term

$$D_{\text{KL}}\!\big(\pi_\theta \| \pi_{\text{ref}}\big) = \mathbb{E}_{y \sim \pi_\theta(\cdot \mid x)}\!\left[\log \frac{\pi_\theta(y \mid x)}{\pi_{\text{ref}}(y \mid x)}\right]$$

### Combined Objective

$$\underset{\pi_\theta}{\max} \; \mathbb{E}_{y \sim \pi_\theta(\cdot \mid x)}\!\left[R(x, y) - \lambda \log \frac{\pi_\theta(y \mid x)}{\pi_{\text{ref}}(y \mid x)}\right]$$

- $\lambda$ controls the trade-off between reward maximization and staying close to $\pi_{\text{ref}}$
- Higher $\lambda$ → stay closer to reference policy
- Lower $\lambda$ → more aggressive reward optimization

---

## 8. Key Takeaways

1. **Alignment prevents undesirable outputs** that pre-training and instruction tuning cannot prevent
2. LLM serves dual roles: as the **policy** and as the **reward model**
3. Reward model is trained via **Bradley-Terry preference model** (log-sigmoid loss)
4. Preferences can come from **humans (RLHF)** or **AI (RLAIF)**
5. Must **stay close to reference policy** to prevent reward hacking and maintain output diversity
6. Combined objective balances **reward maximization** with **KL divergence minimization**
